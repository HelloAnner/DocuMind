# 查询改写 (Query Rewriting)

将用户自然语言提问优化为适合检索的结构化查询，是 Query Pipeline 的第一阶段。直接决定检索召回率的上限。

## 核心职责

- **多轮上下文融合**：结合历史对话消解指代（"那份文档" → "Q3 销售报告"）
- **HyDE**：先让 LLM 生成假想答案段落，用假想答案做向量检索（提升召回 15~20%）
- **Multi-Query**：复杂问题拆成 2-3 个子查询分别检索，结果合并去重
- **术语规范化**：企业黑话/缩写 → 正式术语

## 输出

rewritten_query + keywords + 可选的 hypothetical_answer，并行送入向量检索和 BM25。
